Overview of the project

This is a small, interactive terminal/console game that simulates a simplified version of hand cricket. The user chooses to bat or bowl in the toss, then plays against the computer. Each delivery, the player inputs a number between 1 and 6; the computer randomly selects a number between 1 and 6. If the numbers match, the batting side loses a wicket; otherwise, runs are added to the batting side's total. The match consists of two innings.

Features

The player chooses to bat or bowl at the start (simple toss choice).
Computer randomly bowls or bats using Python's random module.
Wickets and ball counts are tracked.
Two-innings match (each "inning" limited by balls/wickets as per the script).
Simple win/lose output at the end.

Technologies / Tools used

Python 3.13.7 (no external dependencies)
Standard library: random
